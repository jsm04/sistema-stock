# 1. Necesidades del cliente (PO)

El sistema debe resolver problemas típicos de control de inventario en un entorno pequeño/mediano:

1. Registrar productos con atributos básicos (nombre, SKU, categoría, precio, stock actual).
2. Visualizar el estado del stock en tiempo real.
3. Registrar entradas de stock (compras, reposiciones).
4. Registrar salidas de stock (ventas, pérdidas, ajustes).
5. Alertar cuando el stock esté por debajo de un umbral mínimo.
6. Permitir búsqueda y filtrado de productos.
7. Mantener un historial de movimientos de stock (trazabilidad).
8. Soportar múltiples usuarios con roles básicos (admin / operador).
9. Generar reportes simples (stock actual, productos críticos).
10. Evitar inconsistencias (no permitir stock negativo sin autorización).

### 2. Casos de uso

**UC01 – Gestionar productos**

- Crear producto
- Editar producto
- Eliminar producto
- Consultar producto

**UC02 – Consultar inventario**

- Ver listado completo
- Filtrar por categoría / nombre
- Ver detalle de producto

**UC03 – Registrar entrada de stock**

- Ingresar cantidad
- Asociar motivo (compra, devolución, ajuste)

**UC04 – Registrar salida de stock**

- Descontar cantidad
- Asociar motivo (venta, pérdida, ajuste)

**UC05 – Alertas de stock mínimo**

- Detectar umbral bajo
- Notificar al usuario

**UC06 – Historial de movimientos**

- Consultar movimientos por producto
- Consultar movimientos por fecha

**UC07 – Gestión de usuarios**

- Crear usuario
- Asignar rol
- Autenticación

**UC08 – Reportes**

- Reporte de stock actual
- Reporte de productos críticos
